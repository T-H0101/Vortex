# macOS Swift 任务管理监督器 + 活动监视器实施计划（v3）

## 1. 项目目标
在 macOS 本机使用 Swift 构建双窗口悬浮应用，核心目标如下：
- **任务管理监督器窗口**：支持每日任务创建/编辑、完成追踪、到点提醒、未完成追提醒，交互体验参考 Apple 提醒事项。
- **活动监视器窗口**：支持应用级活动监视 + 浏览器标签页/网页级活动监视（Safari + Chrome），并可一键跳转激活目标应用/窗口。
- **统一视觉规范**：毛玻璃 Material 风格 + 悬浮窗口 + 边缘吸附/收纳（类灵动岛交互），整体简洁并隐藏内部技术复杂度。

## 3. 技术栈与工程方案（细化）
### 3.1 语言与框架
- **语言**：Swift 5.10+（随当前 Xcode 稳定版）
- **UI**：SwiftUI（主）+ AppKit（窗口行为、系统级能力补充）
- **数据持久化**：SwiftData（任务、提醒计划、活动索引）
- **系统通知**：UserNotifications（到点提醒与未完成提醒）
- **系统能力**：NSWorkspace / Accessibility / AppleScript(必要时) 用于活动采集与跳转

### 3.2 架构分层（对应“前后端”职责）
- **Presentation（前端层）**：SwiftUI View + ViewModel，负责界面与交互反馈。
- **Domain（业务层）**：任务管理、提醒调度、活动聚合、收纳状态机等核心规则。
- **Infrastructure（本地后端层）**：SwiftData 存储、通知调度器、系统权限与浏览器适配器。
- 说明：本项目是本地桌面应用，无远程后端服务；“后端编写”对应本地业务/数据/系统能力服务层。

### 3.3 目录与模块建议
- `App/`：入口、窗口路由、全局依赖注入
- `Features/TaskSupervisor/`：任务监督窗口（UI + ViewModel + UseCase）
- `Features/ActivityMonitor/`：活动监视窗口（UI + ViewModel + UseCase）
- `Core/Windowing/`：悬浮、吸附、收纳、窗口状态同步
- `Core/Reminder/`：提醒引擎与调度策略
- `Core/ActivityTracking/`：应用活动采集、Safari/Chrome 适配器
- `Data/`：SwiftData 模型、仓储实现
- `Tests/`：单元、集成、UI 测试

## 4. 功能设计与实现边界
### 4.1 任务监督窗口
- 任务 CRUD、每日视图、进度展示、逾期标记
- 提醒策略：固定时间提醒 + 截止后未完成二次提醒 + 次日重排策略
- 交互原则：减少复杂配置入口，默认值可用，错误提示明确

### 4.2 活动监视窗口
- 应用级：前台应用切换事件、最近活跃列表、点击激活跳转
- 浏览器级：Safari + Chrome 当前标签页标题/URL 采集（在用户授权后启用）
- 降级策略：未授权时退化为应用级监视，并给出一次性引导

### 4.3 悬浮与收纳交互
- 双窗口支持悬浮层级、拖拽、边缘吸附、胶囊化收纳
- 悬停/点击展开，离焦后可按策略自动收纳
- 保持“轻量不打扰”：默认不强抢焦点

## 5. Agent 集群划分与协作机制
| Agent | 主要职责 | 输入 | 输出 | 依赖/协作 |
| --- | --- | --- | --- | --- |
| 架构编排 Agent | 拆解任务、定义接口契约、把控节奏 | 产品目标、计划文档 | 模块边界、迭代任务单 | 协调全部 Agent |
| 前端 UI Agent | SwiftUI 页面、毛玻璃样式、窗口交互 | 设计规范、Domain 接口 | 双窗口 UI、交互实现 | 依赖本地后端 Agent、集成 Agent |
| 本地后端 Agent | 数据模型、仓储、提醒调度、业务用例 | 功能规则、权限策略 | 可调用服务层与 UseCase | 协作前端/测试/集成 Agent |
| 活动采集 Agent | 应用活动 + Safari/Chrome 标签页适配 | 系统 API、权限约束 | 活动事件流、跳转能力 | 协作本地后端/测试 Agent |
| 测试 QA Agent | 单元/集成/UI 测试设计与实现 | 功能需求、接口契约 | 测试用例与缺陷清单 | 与全部功能 Agent 并行 |
| 集成发布 Agent | 合并联调、签名配置、构建与打包 | 各模块分支产物 | 可运行构建、发布检查单 | 最终汇总全部 Agent 结果 |

### 5.1 协作节奏（建议）
1. 架构编排 Agent 先完成接口定义和模块契约。
2. 前端 UI Agent 与 本地后端 Agent 并行开发，活动采集 Agent 并行落地浏览器适配。
3. 测试 QA Agent 从第一阶段开始跟进，先写 Domain 单测，再补集成与 UI 自动化。
4. 集成发布 Agent 在每个里程碑执行一次合并联调，最后统一发布收口。

## 6. 测试与质量策略（补充）
### 6.1 测试层次
- **单元测试（XCTest）**：Domain 规则（任务状态流转、提醒重排、收纳状态机）
- **集成测试（XCTest）**：SwiftData 仓储 + Reminder 调度 + Activity 适配器协同
- **UI 测试（XCUITest）**：双窗口打开/切换、任务新增流程、吸附收纳主链路
- **手动验收**：系统权限流程（通知、辅助功能、自动化）和未授权降级路径

### 6.2 质量门禁（每阶段都要满足）
- 可编译、可运行、关键路径可操作
- 新增核心功能必须有对应测试（至少单测或 UI 测试其一）
- 不允许吞错；权限失败/采集失败需可见提示与降级行为

### 6.3 建议命令（工程建立后）
- 构建：`xcodebuild -scheme Vortex -destination 'platform=macOS' build`
- 测试：`xcodebuild -scheme Vortex -destination 'platform=macOS' test`

## 8. 风险与应对
- **macOS 无原生 Dynamic Island**：采用顶部/边缘胶囊收纳浮层替代。
- **浏览器标签页采集受权限约束**：必须实现授权引导 + 应用级降级。
- **系统通知不稳定触达**：增加应用内状态提示作为补充反馈。
- **多 Agent 并行冲突**：严格执行接口契约与阶段集成门禁，减少返工。
